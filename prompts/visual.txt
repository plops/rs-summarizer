Etwaige Dependencies müssen in Cargo.toml geschrieben werden.  Da LLM oft nicht wissen welche version gerade aktuell ist soll als naechstes



 `cargo upgrade --incompatible allow`



ausgefuehrt werden. Dieses Kommando setzt die neueste Versionsnummer im Cargo.toml.

Das Upgrade-Kommando ist in einer frischen Rust installation nicht da und muss mit dem folgenden Kommando erst installiert werden:



 `cargo install cargo-edit`



Bevor eigentlicher Code implementiert wird, sollte das Deepwiki MCP verwendet werden, um Informationen über die Dependencies (sofern diese auf github sind) zu bekommen. Der Lookup im Deep Wiki erfordert eine bestimmte Namensgebung bei der Abfrage für die Dependencies. 

Ich muss noch die GitHub-Organisation angeben:

<github-organization>/<projekt>



Die Tokyo Library findet man auf https://github.com/tokio-rs/tokio. Deshalb Wollte man dem Diebwiki sagen: "Finde informationen zu tokio-rs/tokio". 

Um diese Anfragen zu vereinfachen, Für jede Dependency eine Zeile mit diesen Informationen in eine deps.md Datei geschrieben werden.



Beispeilinhalt deps.md:

